# Contributing

We welcome contributions that improve the course, fix errors, or add practical examples.

## How to contribute

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/module-name-update`.
3. Make your changes.
4. Ensure Markdown files are valid and links are consistent.
5. Open a pull request with a clear description.

## Content guidelines

- Keep the main module content in **Ukrainian**.
- Keep navigation, README, and contribution files in **English**.
- Every code example should be runnable or clearly state dependencies.
- Add Mermaid diagrams where they help understanding.
- Follow the existing module template: `README.md`, `lab.md`, `practice.md`, `homework.md`, `checklist.md`, `resources.md`, `mini-project.md`.

## Code of conduct

Be respectful, constructive, and focused on learning.
