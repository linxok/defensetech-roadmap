# 10. Backend-системи

## Backend для дронів

Backend збирає телеметрію, зберігає місії, управляє флотом, роздає дані GCS і AI-сервісам. Вимагає високої доступності, масштабованості та низької латентності.

## Технологічний стек

- **FastAPI / NestJS** — API-сервіси.
- **PostgreSQL** — реляційні дані (місії, користувачі, журнали).
- **Redis** — кеш, сесії, pub/sub.
- **RabbitMQ** — черги задач.
- **Kafka** — event streaming.
- **WebSockets** — live telemetry для GCS.

## PostgreSQL схема

```sql
CREATE TABLE telemetry (
    id BIGSERIAL PRIMARY KEY,
    drone_id TEXT NOT NULL,
    ts TIMESTAMP NOT NULL,
    lat DOUBLE PRECISION,
    lon DOUBLE PRECISION,
    alt DOUBLE PRECISION,
    battery INTEGER
);
CREATE INDEX idx_telemetry_drone_ts ON telemetry(drone_id, ts);
```

## Redis pub/sub

```python
import redis

r = redis.Redis()
r.publish('telemetry', '{"drone_id": "001", "alt": 100}')
```

## FastAPI endpoint

```python
from fastapi import FastAPI

app = FastAPI()

@app.get("/telemetry/{drone_id}")
async def get_telemetry(drone_id: str):
    return {"drone_id": drone_id, "alt": 120, "battery": 87}
```

## Мініпроєкт

Telemetry pipeline: UDP → Gateway → Queue → Workers → Database → API → Dashboard.
